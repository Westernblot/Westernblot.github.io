docker常用命令

发表于 2018-03-27 | 分类于 docker | | 阅读次数

docker命令
– 进入一个已经正在运行的容器
$ sudo docker ps
$ sudo docker exec -it 775c7c9ee1e1 /bin/bash

docker ps //列出正在运行的容器
docker ps -a //列出所有容器
docker images //查看所有镜像

docker run -p 80:80 –name test -v c:/wampserver/www:/var/www -it 镜像名or镜像id /bin/bash

docker run ：创建一个新的容器并运行一个命令，常用参数:
-i: 以交互模式运行容器，通常与 -t 同时使用；
-t: 为容器重新分配一个伪输入终端，通常与 -i 同时使用
–name=”nginx-lb”: 为容器指定一个名称；
-P 将容器的80端口映射到主机随机端口。
-p 80:80 将容器的80端口映射到主机的80端口
-v 映射主机目录和容器目录

docker cp :用于容器与主机之间的数据拷贝
docker cp /www/runoob 96f7f14e99ab:/www/

centos6.5安装docker报错问题

发表于 2018-03-27 | 分类于 docker | | 阅读次数

uname -r //查看系统内核版本
2.6.32-431.23.3.el6.x86_64

yum命令安装
yum -y install docker-io

报错信息

docker info

Cannot connect to the Docker daemon. Is ‘docker -d’ running on this host?

docker -d

INFO[0000] Listening for HTTP on unix (/var/run/docker.sock)
docker: relocation error: docker: symbol dm_task_get_info_with_deferred_remove, version Base not defined in file libdevmapper.so.1.02 with link time reference

解决方法

应该是libmapper有关，需要升级下这个库:
1、yum-config-manager –enable public_ol6_latest
2、yum install device-mapper-event-libs
（实测只需执行2即可）

安装之后启动 Docker 服务，并让它随系统启动自动加载。

.service docker start
.
.chkconfig docker on

git 学习笔记

发表于 2017-11-15 | 分类于 linux | | 阅读次数

$ git rm tencent
fatal: not removing ‘tencent’ recursively without -r
解决：用git rm –cached来删除文件夹的追踪状态是出现
git rm -r –cached tencent

git push 后仓库没有显示
解决：用 git pull 同步本地和远程
git pull origin master
然后再 git push

$ git push
Everything up-to-date
远程么显示
解决：添加git commit后再git push

添加文件:
git add tencent
git commit -m “XXX”
git push

删除仓库文件:
git rm -r –cached tencent
git commit -m “移除”
git push
