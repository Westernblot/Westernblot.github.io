cordova 自定义插件开发

发表于 2016-11-28 | 分类于 跨平台移动开发 | | 阅读次数

插件开发
6.1 pluman的安装
npm install -g plugman
6.2 plugman安装完之后就可以创建一个插件了cordova plugin
plugman create –name –plugin_id –plugin_version [–path ] [–variableNAME=VALUE]
参数:
pluginName: 插件名字
pluginID: 插件id, egg : coolPlugin
oversion: 版本, egg : 0.0.1
directory:一个绝对或相对路径的目录，该目录将创建插件项目
variable NAME=VALUE: 额外的描述，如作者信息和相关描述
egg : plugman create –name CoolPlugin –plugin_id me-plugin-coolPlugin –plugin_version 0.0.1
遵循规范的话，一般在src目录下新建android目录，然后在android目录下新建类，如下图

plugin.xml

coolPlugin.js

ShowToast.java

插件的安装
我的插件所在的路径是F:\CoolPlugin
执行插件安装命令 cordova plugin add F:\CoolPlugin
执行完之后你就发现插件已经安装上去了
如果你想卸载插件的话，执行cordova plugin remove 你的plugin_id
egg:cordova plugin remove coolPlugin
